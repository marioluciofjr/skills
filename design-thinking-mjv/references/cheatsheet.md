# Cheatsheet — Design Thinking: Inovação em Negócios

## Processo DT: Visão Geral

```
IMERSÃO ──────────────────────────────────────────────────────────────────────
Preliminar: Reenquadramento + Pesquisa Exploratória + Pesquisa Desk
Profundidade: Entrevistas + Cadernos + Sessão Generativa + Sombra + Um Dia na Vida

ANÁLISE E SÍNTESE ────────────────────────────────────────────────────────────
Cartões de Insights → Diagrama de Afinidades → Mapa Conceitual
Critérios Norteadores | Personas | Mapa de Empatia | Jornada do Usuário | Blueprint

IDEAÇÃO ──────────────────────────────────────────────────────────────────────
Brainstorming → Workshop de Cocriação → Cardápio de Ideias → Matriz de Posicionamento

PROTOTIPAÇÃO ─────────────────────────────────────────────────────────────────
Papel | Volume | Encenação | Storyboard | Protótipo de Serviços
[ciclo: formular questões → criar → testar → avaliar → repetir]
```

> As fases NÃO são lineares. Prototipação pode ocorrer durante a Imersão. Ideação permeia todo o processo.

## Tabela de Seleção de Técnicas de Pesquisa

| Objetivo | Técnica recomendada |
|---|---|
| Entender o contexto inicial | Pesquisa Exploratória |
| Mapear tendências e análogos | Pesquisa Desk |
| Coletar histórias e significados | Entrevistas |
| Capturar comportamentos não verbalizados | Sombra |
| Gerar empatia profunda | Um Dia na Vida |
| Revelar aspectos tácitos e latentes | Sessão Generativa |
| Coletar dados remotos ou de longo período | Cadernos de Sensibilização |

## Tabela de Seleção de Protótipo

| Tipo de solução | Protótipo indicado | Fidelidade inicial |
|---|---|---|
| Interface gráfica / app | Em Papel (wireframe) | Baixa |
| Produto físico | Modelo de Volume | Baixa a média |
| Interação de serviço humano | Encenação | N/A (ao vivo) |
| Narrativa / conceito | Storyboard | Baixa |
| Sistema de serviço completo | Protótipo de Serviços | Média a alta |

## 4 Princípios do Brainstorming
1. Qualidade pela quantidade
2. Evitar julgar (adiar avaliação)
3. Ideias ousadas são bem-vindas
4. Combinar e aprimorar (100% colaborativo)

## Mapa de Empatia: 6 Quadrantes
| Quadrante | Pergunta |
|---|---|
| Vê | O que enxerga no ambiente? |
| Ouve | Como o ambiente o influencia? |
| Pensa/Sente | O que está na cabeça? |
| Diz/Faz | Como se comporta publicamente? |
| Dificuldades | Quais são seus empecilhos? |
| Conquistas | O que deseja alcançar? |

## Anti-patterns Críticos

| Anti-pattern | Consequência |
|---|---|
| Tratar o processo DT como linear | Perde a riqueza das iterações entre fases |
| Resolver problema com mesmo pensamento que o criou | Soluções incrementais, não inovadoras |
| Persona média (não extrema) | Soluções medíocres sem diferenciação |
| Brainstorming sem insumos de Imersão | Ideias desconectadas da realidade do usuário |
| Julgamento durante brainstorming | Mata as ideias mais inovadoras prematuramente |
| Equipe homogênea | Perde perspectivas cruzadas; converge no óbvio |
| Prototipação tardia | Aprendizado caro; erros identificados tarde |
| Alta fidelidade prematura | Investimento em direção não validada |

## Pesquisa DT vs. Pesquisa de Mercado

| Dimensão | Pesquisa DT | Pesquisa de Mercado |
|---|---|---|
| Amostragem | Perfis extremos | Representação estatística |
| Tipo de dado | Comportamentos, emoções, narrativas | Opiniões e comportamentos declarados |
| Método | Conversas semiestruturadas | Questionários estruturados |
| Objetivo | Inspirar; identificar oportunidades latentes | Prever comportamentos; validar hipóteses |
